Проблема "Простановка приоритетов по умолчанию"

Есть алгоритм простановки попарных приоритетов обработчиков (сначала обработчик-писатель, потом обработчик-читатель). Алгоритм пришлось отключить, по следующей причине:

- есть обработчики, для которых приоритеты проставлялись вручную и отличались от значений по умолчанию;

- когда в одном из обрабочиков пары меняли читаемые-изменяемые данные, информация о приоритетах сбрасывалась (и это правильно);

- запускали алгорит простановки приоритетов по умолчанию, и он ставил для этих обработчиков значения по умолчанию (запускали не те люди, которые ставили приоритеты, отличные от значений по умолчанию)

Решение

- в регистре с приоритетами сохраняем признак, что приоритет настроен вручную и отличается от дефолтного - в этом случае приоритеты не буду заполняются значениями по умолчанию.
